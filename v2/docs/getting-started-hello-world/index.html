Serving requests#

Include#

#include "pistache/endpoint.h"

Hello World#

Requests received by Pistache are handled with an Http::Handler.

Letâ€™s start by defining a simple HelloHandler:

using namespace Pistache;
class HelloHandler : public Http::Handler {
public:
HTTP_PROTOTYPE(HelloHandler)
void onRequest(const Http::Request& request, Http::ResponseWriter response) {
response.send(Http::Code::Ok, "Hello, World");
}
};

Handlers must inherit the Http::Handler class and at least define the onRequest member function.

A handler must also define a clone() member function. Simple handlers can use the special HTTP_PROTOTYPE macro, passing in the name of the class. The macro will automatically expand to a default implementation of the clone() member function.

Final touch#

After defining the handler, the server can now be started:

int main() {
Net::Address addr(Net::Ipv4::any(), Net::Port(9080));
auto opts = Http::Endpoint::options().threads(1);
Http::Endpoint server(addr);
server.init(opts);
server.setHandler(std::make_shared<HelloHandler>());
server.serve();
}

For simplicity, you can also use the listenAndServe helper function that will automatically create an endpoint and instantiate your handler:

int main() {
Http::listenAndServe<HelloHandler>("*:9080");
}
Use more serving threads

By default, listenAndServe will only use 1 thread. You can tweak that by passing an Options argument to the function:

auto opts = Http::Endpoint::options().threads(std::thread::hardware_concurrency());
Http::listenAndServe<HelloHandler>("*:9080", opts);

And thatâ€™s it, now you can fire up your favorite curl request and observe the final result:

curl http://localhost:9080/
Hello, World
